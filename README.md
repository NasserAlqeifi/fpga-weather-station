# FPGA Weather Station

A real-time environmental monitoring system implemented on an Intel Cyclone V FPGA. The system integrates a Nios II softcore processor with multiple sensors to measure temperature, humidity, and barometric pressure. Readings are displayed on an I2C LCD, and configurable thresholds trigger LED and display alerts when environmental conditions exceed user-defined limits.

---

## Overview

This project demonstrates a complete embedded SoC design on an FPGA, combining RTL hardware description with embedded C firmware. Two sensors provide complementary environmental data: the DHT11 handles temperature and humidity over a single-wire protocol, while the HP206C barometric pressure sensor communicates over I2C. A four-line character LCD shows live readings, and four push buttons let the user navigate between display screens and adjust alert thresholds at runtime.

The hardware design was built using Intel Platform Designer (Qsys) and compiled with Quartus Prime 18.1.0. The firmware runs on a Nios II/e softcore processor at 50 MHz using the Nios II Hardware Abstraction Layer (HAL).

---

## Features

- Dual-sensor temperature averaging - combines DHT11 and HP206C readings for improved accuracy
- Humidity monitoring via DHT11 single-wire interface with checksum verification
- Barometric pressure monitoring via HP206C over I2C
- Four-line I2C character LCD with real-time sensor data display
- Four navigation screens: sensor readings, temperature limit, pressure limit, and humidity limit
- Adjustable alert thresholds for temperature, pressure, and humidity, configured with LEFT/RIGHT buttons
- LED alert system with synchronized LED blink and display flash when any threshold is exceeded
- Nios II/e soft processor running a bare-metal C application at 50 MHz
- Bidirectional open-drain I2C interface implemented correctly at the RTL wrapper level

The top-level module `weather_station_I2C` handles the bidirectional I2C bus: SDA and SCL are declared as `inout` ports with open-drain drivers controlled by output-enable signals from the Qsys system. The Qsys system connects the Nios II processor to all peripherals over an Avalon-MM interconnect generated by Platform Designer.

## Module Descriptions

### `weather_station_I2C.v` - Top-Level RTL Wrapper

The top-level Verilog module. Implements correct bidirectional open-drain logic for I2C SDA and SCL, converting the separate `sda_in / sda_oe` and `scl_in / scl_oe` signals from the Qsys system into true `inout` FPGA I/O pins. Instantiates the Platform Designer-generated `weather_station` system.

### `weather_station` - Platform Designer System

Generated by Intel Platform Designer (Qsys). All peripherals connect to the Nios II processor over the Avalon-MM interconnect fabric.

| Component | Type | Description |
|-----------|------|-------------|
| Nios2 | Processor | Nios II/e softcore, 50 MHz, tiny implementation |
| SRAM | Memory | On-chip SRAM for code and data |
| I2C | Interface | Altera Avalon I2C master controller |
| UART | Debug | JTAG UART for `printf` output and debugging |
| timer_0 | Timer | System timer supporting `usleep()` |
| DHT11 | PIO | Single bidirectional GPIO for DHT11 data line |
| LED_Green | PIO | Output PIO driving the green LED |
| LED_RED | PIO | Output PIO driving the red LED |
| Down / Up / Left / Right | PIO | Input PIOs for the four navigation buttons |

### `weather_station_firmware.c` - Nios II Firmware

The complete embedded C application organized into functional sections:

**DHT11 Driver** - Software bit-bang implementation of the DHT11 single-wire protocol. Generates the start pulse (18 ms low), detects the sensor response, reads 40 bits of data, and verifies the checksum. Returns integer temperature (deg C) and humidity (%).

**HP206C Driver** - I2C communication with the HP206C barometric pressure sensor. Sends the ADC conversion command (0x50), waits for conversion, reads 6 bytes of raw pressure and temperature data, and converts the 24-bit fixed-point values. Returns integer temperature (deg C) and pressure (hPa).

**LCD Driver** - Initializes the 4-line I2C character display, sends commands and data bytes using the Altera Avalon I2C HAL, and provides `write_string()` with DDRAM address selection for each of the four lines.

**Main Loop** - Reads button states with debounce, navigates between the four screens, reads both sensors, averages temperature, formats output strings, writes to the display, and triggers the alert routine (5x LED blink cycle) when any threshold is exceeded.

## Hardware Requirements

- Intel/Altera Cyclone V development board with device **5CSEMA4U23C6** (e.g., DE1-SoC or compatible)
- **DHT11** temperature and humidity sensor - data pin connected to **PIN_U14**
- **HP206C** barometric pressure and temperature sensor - connected to I2C bus (address `0x76`)
- **I2C LCD display** - 4-line character display with SSD1311-compatible controller, I2C address `0x3C`
- Four push buttons connected to: UP (`PIN_U13`), DOWN (`PIN_AG9`), LEFT (`PIN_AG8`), RIGHT (`PIN_AG10`)
- Green LED on `PIN_AF17`, red LED on `PIN_AH8`
- Active-low reset button on `PIN_AH17`
- I2C SDA on `PIN_AH9`, I2C SCL on `PIN_AG11` (3.3V LVTTL)
- 50 MHz system clock on `PIN_V11`

## Software Requirements

- **Intel Quartus Prime 18.1.0 Lite Edition** (or a compatible later version) for FPGA synthesis, placement, and routing
- **Intel Platform Designer (Qsys)** - bundled with Quartus Prime - for viewing and regenerating the SoC system
- **Nios II Software Build Tools for Eclipse** - for building the embedded C application and BSP
- **Nios II Command Shell** - for downloading the ELF binary to the target

## Getting Started

### Step 1 - Open the Quartus Project

Open `weather_station/weather_station.qpf` in Quartus Prime 18.1.0.

### Step 2 - Compile the Hardware

Run a full compilation: Processing > Start Compilation. Quartus synthesizes, fits, and assembles the design. The compiled programming file `weather_station.sof` is written to `weather_station/output_files/`.

### Step 3 - Program the FPGA

Connect the board via USB-Blaster. Open the Quartus Programmer (Tools > Programmer), select the `.sof` file, and click Start to configure the Cyclone V FPGA.

### Step 4 - Build the Embedded Software

Open Nios II SBT for Eclipse. Import the workspace from `weather_station/software/`. Build the BSP project (`code_bsp`) first, then build the application project (`code`). The compiled ELF binary appears in `code/Debug/` or `code/Release/`.

### Step 5 - Run the Application

With the FPGA programmed, use Run > Run As > Nios II Hardware in Eclipse, or use `nios2-download` from the Nios II Command Shell to load the ELF and start execution. The LCD will initialize and begin displaying live sensor data within a few seconds.
